# v0.2.4 更新说明

## 重点改动

### 采购流程
- 删除「接单 / 财务审批」两个环节，运营提交即进入采购处理；采购"提交付款"直接进入待付款
- 新增 1688 订单 / 消息事件**自动同步调度器**：每 10 分钟扫描未绑定订单号的 PO 自动回填，并补跑 unmatched 消息事件
- 1688 取消订单加**死单兜底**：远端订单不存在时自动本地清绑（`orphan_cleared`），不再卡死
- 已取消 / 死单 PO 现在可以删除（之前规则太严无法清理）

### 1688 数据正确性修复
- JSON 解析保留 19 位 Long 订单号精度（之前 `...43584` 被四舍五入成 `...43600` 导致所有后续 1688 接口"订单不存在"）
- `createRefund / addOrderMemo / addOrderFeedback / confirmReceiveGoods` 顶层补 `orderId` / `orderEntryIds` / `disputeRequest` / `goodsStatus` / `applyPayment`（按分计），符合 1688 API 校验
- `orderEntryIds` 推导兼容 `subItemID` / `subItemIDString` / `productSnapshotUrl` 多种来源
- `updatePurchaseOrderFrom1688Snapshot` 改成 merge 而非覆盖，防止 add_memo 等动作擦掉订单详情
- 11 种常见 1688 业务错误码（ORDER_NOT_EXIST / ACCESS_TOKEN_EXPIRED / SYSTEM_ERROR …）映射成可操作中文提示

### 入库流程
- 付款确认时**自动桥接**生成入库单（`erp_inbound_receipts`），仓库中心立刻可见
- 入库单号改 6 位纯数字自增（`000001`、`000002` …）
- 入库流程合并成单按钮「入库」：缺数量按 PO 自动充满 → 状态推到已入库待质检 → 建库存批次 → 商品资料的"实际库存数"立刻更新
- 「按实数入库」：弹 Modal 逐行填实收 / 破损，差异自动算"短少 N / 多到 +N"

### UI / 体验（聚水潭风格）
- 顶部 5 步流程节点条（新建采购单 → 推送1688 → 提交付款 → 付款发货 → 入库完成）
- 表格行高从 90px 压到 32-40px，一屏从 5 行变 18-20 行
- 状态 Tag 上色：橙=待办、蓝=进行、绿=完成、红=异常、灰=取消
- 整合 1688 支付成单按钮，点击直接调系统默认浏览器打开 1688 收银台
- 推送 1688 单时弹地址选择 Modal（默认地址自动用，多地址才弹）
- 仓库中心主从结构：点入库单行 → 下方 Tab 显示「库存批次 / 物流 / 操作信息」
- 采购单行的「物流」列：显示 1688 物流公司 + 可复制运单号
- 1688 单号点击 → 复制 + 浏览器打开 1688 工作台

### 性能
- IPC 默认软超时（workbench 30s / action 60s），UI 不再卡死
- 5 个状态切换按钮 + 删除 + 推 1688 走乐观更新，点击瞬间 UI 反馈
- 关闭前端 import_1688_orders 自动轮询（服务器调度器已接管）
- 页面打开走异步加载：有缓存先显示老数据，新数据后台拉
- broadcast 触发的 loadData 延后到 1500ms，给乐观更新留时间

### 错误处理
- 二次确认仅保留 destructive 操作（删除 PR/PO、免密支付、取消1688）
- 「确认收货 / 退款」业务态拒绝时给出具体原因，而不是 generic toast

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下，下载新版自动安装。

## 服务器侧

服务器端代码已经一并更新到同一 commit；新调度器已在服务器运行（`[order-sync] scheduler started, interval=10min`）。
