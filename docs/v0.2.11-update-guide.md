# v0.2.11 更新说明

## 重点改动：IPC 通用超时统一提到 120s + 默认跳过 1688 mtop

### 0.2.10 仍有的问题
- 「同步规格」按钮点击后报 `IPC erp:purchase:action 调用超时 (60s)`：refresh_1688_product_detail 不在 slowActions 列表里，沿用 60s 默认
- 1688 mtop 网页接口对裸 fetch 反爬/慢，AbortController 在 socket connect 阶段经常不生效

### 0.2.11 修复
1. **preload.cjs**：`erp.purchase.action` 默认 IPC 软超时从 60s 提到 **120s**——覆盖所有未列入 slowActions 的 1688 / alphashop 相关 action
2. **PurchaseCenter runAction**：slowActions 改成 regex 匹配（`/1688|alphashop/i`），所有名字含 1688/alphashop 的 action 自动 120s
3. **buildClientImageSearchMockResults**：默认跳过客户端 1688 mtop 预搜（`ERP_CLIENT_IMAGE_PRESEARCH=1` 可恢复）
4. **source1688ImageAction (主控端)**：默认跳过 1688 mtop 网页图搜（`ERP_SERVER_WEB_IMAGE_SEARCH=1` 可恢复）

### 效果
- 「以图搜款」 / 「同步规格」 / 「推送 1688 下单」/「解析规格」等 1688 相关动作：**最坏 25s 内**返回结果（alphashop 兜底，1-2 秒就回）
- 不再撞 60s 软超时

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下。

## 服务器侧

主控端 `erp.temu.chat` 已同步部署，无需额外操作。
