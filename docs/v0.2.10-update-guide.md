# v0.2.10 更新说明

## 重点改动：以图搜款超时彻底修复（hotfix，跳过 0.2.9）

### 0.2.8 / 0.2.9 已知问题

点「以图搜款」60 秒后报「IPC erp:purchase:action 调用超时 (60s)」。0.2.9 hotfix 只缩短了 1688 mtop 内部 fetch 超时，但客户端 IPC 软超时（preload 默认 60s）还是优先触发，结果体验没改善。

### 0.2.10 修复

PurchaseCenter 的 `runAction` 智能路由 IPC 软超时——已知慢动作（`source_1688_image` / `push_1688_order` / `preview_1688_order` / `preview_1688_url_specs`）默认用 **120s** 软超时，其它继续 60s。

效果：
- 「以图搜款」最坏 25s（1688 mtop fetch）→ 自动 fallback 到主控端 alphashop（遨虾）→ 25s 内返回候选商品
- IPC 软超时不再提前砍断 fallback 链
- 推送 1688 下单 / preview 等其他慢链路一并享受

### 同期一起带的 0.2.9 修复

`run1688WebImageSearch`（client + server 两端）裸 fetch 1688 mtop 接口的超时从 120s 降到 25s。失败后请求继续到主控端用 alphashop 兜底（已配置 credentials）。

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下，下载新版自动安装。

## 服务器侧

主控端 `erp.temu.chat` 已经同步部署 0.2.9 的 25s 修复，无需额外操作。
