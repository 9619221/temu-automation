# v0.2.9 更新说明

## 重点改动：以图搜款超时大幅缩短（hotfix）

### 0.2.8 已知问题
点「以图搜款」后页面持续转圈，最坏需要等 8 分钟才会失败。

### 根因
`source_1688_image` 在 client 端 `buildClientImageSearchMockResults` 和主控端 `source1688ImageAction` 都通过 `run1688WebImageSearch` 直接 fetch 1688 mtop 接口（warmCnaCookie / acquireMtopToken / uploadImage / searchOffers 共 4 步）。1688 对没登录 cookie 的裸 fetch 容易反爬或响应慢，原 120s 超时把整个 UI 卡死。

### 修复
两处调用点超时降到 **25s**。失败后自动 fallback 到主控端的 alphashop（遨虾）接口，alphashop 已配置 credentials 能稳定返回候选商品。

效果：
- 「以图搜款」最坏 **25 秒**就能拿到结果（成功或者通过 fallback 路径返回）
- 不会再出现页面长时间转圈卡死

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下，下载新版自动安装。

## 服务器侧

主控端 `erp.temu.chat` 已经同步升级，无需额外操作。
