# v0.2.12 更新说明

## 重点改动：采购中心刷新（workbench）IPC 默认超时 30s → 120s

### 0.2.11 仍有的问题
- 「以图搜款」点完之后弹「IPC erp:purchase:workbench 调用超时 (30s)」红色提示
- 主控端日志显示 source_1688_image 实际 1.9s 就完成，但客户端模式下采购中心紧接着会再调一次 `/api/purchase/workbench` 拿最新候选数据
- 客户端通过 https 走 caddy 反代主控端，30s 软超时在 SQL 聚合 + 网络抖动时不够用

### 0.2.12 修复
1. **preload.cjs**：`erp.purchase.workbench` 默认 IPC 软超时从 30s 提到 **120s**
2. **getPurchaseWorkbenchRuntime**：客户端模式 remoteRequest 显式带 `timeoutMs: 120000`，避免底层 https socket 沿用 8s 默认
3. **lanServer /api/purchase/workbench**：增加耗时日志（sql=…ms bodyLen=… writeJson done=…ms），后续可以从日志看出是否真的慢

### 效果
- 「以图搜款」「同步规格」「推送 1688 下单」等动作完成后，采购中心自动刷新不再撞 30s 超时
- 即使主控端聚合较慢（很多 PR/PO），最坏 120s 也能等到响应

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下。
