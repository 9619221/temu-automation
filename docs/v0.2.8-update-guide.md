# v0.2.8 更新说明

## 重点改动：多 1688 采购账号 + 推单账号选择

### 业务场景
- 同一公司可以绑定多个 1688 买家账号（比如「公司主账号」「代采账号」并存）
- 每个 Temu 店铺指定默认 1688 采购账号
- 推单时按「显式选 → 店铺默认 → 公司第一个」三级取凭据

### 主控端 web UI 改动（`https://erp.temu.chat/1688`）
- 之前：单账号表单（覆盖式更新）
- 现在：账号列表 + 「+ 新增 1688 采购账号」按钮
- 每张账号卡片可：粘贴新 access_token / 改 label / 刷新 token / 删除
- 删除占用拦截：被任意 Temu 店铺设为默认时拒绝并提示具体占用列表

### 客户端 ERP 改动
- **店铺管理**：每个 Temu 店铺新增「默认 1688 采购账号」下拉
- **采购中心 → 推送 1688 下单**：智能流程
  - 0 个可用 1688 账号 → 报错引导去配置
  - 1 个账号 → 自动用，不弹 Modal
  - 多账号 + 店铺有默认 → 用默认，不弹
  - 多账号无默认 → 弹「选择 1688 采购账号」Modal 让用户点选
- 主控端旧版本静默回落：服务器没升级时按 0.2.7 行为单账号推单，不会被 toast 挡住

### 数据模型
- DB migration 026：`erp_1688_auth_settings` 加 `label` / `status`；`erp_accounts` 加 `default_1688_purchase_account_id`
- 自动跑，不影响现有数据

## 升级方式

桌面客户端：「设置 → 应用更新 → 检查更新」点一下。

## 服务器侧

⚠️ **主控端必须 git pull 到 master 并重启服务才能用上多账号功能**：

```bash
ssh root@erp.temu.chat
cd /opt/temu-automation
sudo -u ubuntu git pull origin master
systemctl restart temu-erp.service
curl -sS http://127.0.0.1:19380/health   # 应返 ok
```

migration 026 服务启动时自动跑。

如果客户端升级了 0.2.8 但主控端还没升，多账号下拉会显示「暂无可用账号」、推单会回落到旧的单账号行为（不报错，但没有多账号选择能力）。

## 给所有客户/同事的话术

> v0.2.8 上线了「多 1688 采购账号」能力。如果你们公司想绑多个 1688 买家账号给不同 Temu 店铺采购：
> 1. admin 用户去主控端 `https://erp.temu.chat/1688` 用「+ 新增账号」表单加新 1688 账号
> 2. 客户端「店铺」里给每个 Temu 店铺指定默认 1688 采购账号
> 3. 推单时自动按默认走；多账号无默认时弹 Modal 让你选
>
> 如果只用一个 1688 账号，行为跟 0.2.7 完全一致——这个改动不破坏任何现有用法。
